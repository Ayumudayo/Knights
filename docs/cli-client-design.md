# CLI 채팅 클라이언트 설계(개요)

## 범위/목표(후순위)
- 서버 코어 안정화 이후 구축. 텍스트 기반 CLI로 채팅/테스트/관리 기능 제공.

## 기능
- 연결/로그인/채널 조인/메시지 전송/브로드캐스트 수신.
- 스크립트/매크로(선택): 반복 시나리오 실행.
- 프로파일링 지원: RTT 측정, 전송/수신 속도.
- 입출력 분리 UI: 출력과 입력을 명확히 분리하여 사용성 보장.

## 현재 MVP 구현 요약
- 공통 프레이밍 사용(14B 헤더). 접속 시 서버 HELLO 수신 후 동작.
- 백그라운드 수신 스레드로 모든 수신 패킷을 즉시 로그/표시.
- 주기적 PING 송신(8초 간격)으로 서버의 read 타임아웃을 피함.
- 지원 슬래시 명령:
  - `/login <user>`: 로그인, 서버는 `LOGIN_RES` 반환 및 `lobby` 자동 입장.
  - `/join <room>`: 방 입장. 서버는 해당 방에 시스템 브로드캐스트로 입장 알림.
  - `/leave [room]`: 방 퇴장. 서버 시스템 브로드캐스트로 퇴장 알림.
  - `/rooms`: 서버에 방 목록 요청(시스템 브로드캐스트로 응답은 요청자에게만 표시).
  - `/who [room]`: 대상 방의 사용자 목록 요청(시스템 브로드캐스트로 응답은 요청자에게만 표시).
  - `/say <text>` 또는 일반 텍스트: 현재 방에 채팅 전송 → 방 멤버에게 브로드캐스트.
- 에러 표시: `MSG_ERR` 수신 시 코드명과 메시지를 표기(예: UNAUTHORIZED/NO_ROOM/NOT_MEMBER 등).

## 슬래시 명령 및 자동완성
- 형태: 슬래시(`/`) 프리픽스 기반. 예) `/connect`, `/login`, `/join`, `/say`, `/whisper`, `/rooms`, `/who`, `/ping`, `/quit`, `/help`.
- 자동완성 힌트: Codex CLI, Gemini CLI 유사 UI 제공.
  - Tab: 후보 자동완성, Ctrl+Space: 힌트 창 토글.
  - 고스트 텍스트(미리보기)와 후보 리스트(dropdown) 동시 제공.
  - 인자 힌트: 명령 스펙에 따라 자리표시자와 타입/설명 노출.
- 컨텍스트 인식: 현재 접속/룸/최근 사용자 목록을 기반으로 동적 후보 생성.

### 랭킹/필터링 규칙(초안)
- prefix 우선 > subsequence 매칭 > fuzzy(에디트 거리) 보정.
- 최근 사용/성공률 가중치(세션 로컬 히스토리) 적용.
- 위험 명령(`/quit`, `/disconnect`)은 하단 배치 및 확정 시 확인 프롬프트.

### 힌트 포맷 예시
- `/say <message...>`: `message`는 UTF-8 텍스트, 길이 1~400자.
- `/join <room>`: `room`은 알파뉴메릭/`-_/` 포함 가능, 길이 1~64.

### 출력/입력 분리 세부
- 출력은 ring buffer(예: 2000라인) 유지. 신규 출력은 입력줄 위로 렌더, 입력줄은 즉시 리렌더.
- 출력 폭/색상: ANSI 컬러(Windows 콘솔 API 병행) 지원, 링크/멘션 하이라이트.
- 스로틀링: 초당 최대 렌더 라인 제한으로 입력 방해 최소화.

### 네트워킹/스레딩
- Net I/O 수신 스레드 1개 + 메인 스레드(입력/송신). 간단한 공유 플래그로 종료 동기화.
- 프로토콜/프레이밍은 서버와 동일(`protocol.md`) 재사용.

### 스크립트/매크로
- 단순 매크로 파일 실행: 각 라인별 명령/지연(ms) 지정.
- 변수 치환: `${user}`, `${room}` 등.

### 명령 레지스트리(예시)
```json
{
  "commands": [
    {"name": "/connect", "args": ["<host>", "<port:int>"], "desc": "서버에 연결"},
    {"name": "/login",   "args": ["<user>", "<token>"],     "desc": "로그인"},
    {"name": "/join",    "args": ["<room>"],                "desc": "룸 입장"},
    {"name": "/say",     "args": ["<message...>"],          "desc": "메시지 전송"},
    {"name": "/whisper", "args": ["<user>", "<message...>"],"desc": "귓속말"},
    {"name": "/rooms",   "args": [],                         "desc": "룸 목록"},
    {"name": "/who",     "args": ["[room]"],                "desc": "사용자 목록"},
    {"name": "/ping",    "args": [],                         "desc": "핑"},
    {"name": "/quit",    "args": [],                         "desc": "종료"}
  ]
}
```

## 아키텍처
- Net: Boost.Asio 클라이언트 소켓, 동일한 `protocol.md` 재사용.
- UI: 입력/출력 분리 구조
  - 입력부(Input Pane): 라인 에디터(replxx/linenoise-ng 등) 사용, 히스토리/자동완성/힌트 제공.
  - 출력부(Output Pane): 별도 스레드가 스크롤백 버퍼에 누적, 새로운 로그/채팅은 입력줄을 깨뜨리지 않고 위로 렌더.
  - 출력 멀티플렉서: 입력줄 유지(리렌더) + 출력 라인 추가를 조정(ANSI 시퀀스, Windows 콘솔 API 동시 지원).
- 플러그: 메시지 핸들러 테이블(서버와 동일한 구조), slash 명령 파서/실행기.

## 테스트
- 에코/채팅/핑-퐁 시나리오 자동화 스크립트.
- 자동완성 유닛 테스트: 명령 레지스트리 기반 후보 생성/정렬/필터링.
- 스트레스: 빠른 출력이 입력에 영향 주지 않는지 확인(수백 라인/초).

## 구현 선택지
- C++: replxx + Boost.Asio(권장, 서버와 도구 체계 통일).
- 대안: Rust(crossterm + rustyline), Go(tcell + liner). 기능/생태계 감안하여 결정.
