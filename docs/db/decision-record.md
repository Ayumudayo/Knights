# ADR: 데이터베이스 및 보안 선택

- 상태: 승인됨
- 날짜: 2025-09-12

## 배경
- 신뢰성 높은 ACID RDB가 필요하며, 풍부한 SQL 기능을 원한다.
- 서버는 비동기 네트워킹을 사용하므로, DB I/O는 메인 `io_context`를 블로킹하면 안 된다.
- 보안은 복잡성을 과도하게 늘리지 않는 선에서 더 강한 방식을 선호한다.
- 확장성은 우선 설정 기반으로 달성하고, MSA/아웃박스는 이후 단계로 고려한다.

## 결정
- 데이터베이스: PostgreSQL 16
- 드라이버: libpqxx(C++17), Prepared Statement 사용, 초기에는 ORM 미도입
- 비밀번호 해시: Argon2id(선호), 보안 파라미터 충분히 강하게
- 로컬 개발 DB: 기본은 외부 Postgres 사용, Docker Compose는 선택 사항
- 확장성: 풀 크기/타임아웃/리밋 등 설정값으로 튜닝, 초기에는 코드 레벨 샤딩 미도입
- 구현 정책: 사용자 승인 전까지 문서화만 진행, 코드 변경 없음

## 결과
- `core`에 저장소 인터페이스(SPI), `server/storage/postgres`에 Postgres 어댑터를 둔다.
- 마이그레이션은 버전형 SQL 파일로 관리, 경량 러너로 적용한다.
- 보안 수준 개선(토큰 해시 저장, Argon2id), DB 롤로 최소 권한 운영.
- 온보딩은 단순: Docker 유무와 관계없이 `DB_URI`로 접속 가능.

## 대안 검토
- MySQL/MariaDB: 충분하나 고급 SQL/CTE 측면에서 선호도 낮음
- SQLite: 단일 노드 개발엔 좋으나 동시성/기능 제약 존재
- ORM: 복잡성/임피던스 증가, 초기에는 명시적 SQL 선호
