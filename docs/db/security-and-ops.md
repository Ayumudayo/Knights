# 보안, 토큰, 운영

원칙: 더 강한 보안 프리미티브와 최소 권한.

## 비밀번호
- 알고리즘: Argon2id
- 권장 파라미터(개발): `m=64MiB`, `t=3`, `p=1`
- 권장 파라미터(운영 기본): `m=256MiB`, `t=3`, `p=2`(SLO에 맞춰 튜닝)
- 해시 문자열만 저장, 평문은 저장 금지

## 세션 토큰
- 128비트 랜덤 토큰 생성(예: `random_device` + `RAND_bytes` 상응)
- DB에는 `token_hash`(예: SHA-256)만 저장, 제시된 토큰을 해시 비교
- `expires_at` 포함, `revoked_at` 옵션 지원
- 민감 작업 시 토큰 회전

## DB 롤
- 앱 전용 최소 권한 사용자 생성(앱 스키마 한정)
- 리드 레플리카 사용 시 읽기 전용 사용자 분리 가능

## 설정/시크릿
- `DB_URI`는 환경변수/설정 파일로 주입, 시크릿 커밋 금지
- 환경에 따라 `DATABASE_SSLMODE=prefer|require` 고려
- Argon2id 파라미터는 향후 튜닝을 위해 설정화

## 백업/보관
- 논리 백업(pg_dump) 일일, 물리 베이스 백업 주기적 실행(RPO/RTO에 따름)
- 복구 테스트 정기 수행

## 관측
- 메트릭: 풀 사용률, 획득 지연, 쿼리 지연, 에러율, 마이그레이션 버전
- 로그: SQLSTATE/정규화 에러 카테고리 포함, PII는 로그 금지
